## 📦 Open Source — flutter_clean_starter

> **Skip the boilerplate. Start building Flutter apps the right way.**

[![pub.dev](https://img.shields.io/pub/v/flutter_clean_starter?label=pub.dev&color=0175C2)](https://pub.dev/packages/flutter_clean_starter)
[![Pub Points](https://img.shields.io/pub/points/flutter_clean_starter?color=2E7D32)](https://pub.dev/packages/flutter_clean_starter/score)
[![Dart SDK](https://img.shields.io/badge/Dart-%3E%3D3.0.0-0175C2)](https://dart.dev)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://pub.dev/packages/flutter_clean_starter/license)

A CLI tool that scaffolds a **production-ready Flutter project in seconds** from a single interactive command.

```bash
dart pub global activate flutter_clean_starter
flutter_clean_starter create
```

**What it generates:**
- ✅ Full Clean Architecture folder structure (data / domain / presentation)
- ✅ Your choice of state management — BLoC, Cubit, or Riverpod
- ✅ Dio HTTP client with interceptor + PrettyDioLogger pre-configured
- ✅ GetIt + Injectable dependency injection, ready for build_runner
- ✅ Optional Flutter Flavors setup (dev / staging / prod)
- ✅ Runs `flutter pub get` automatically

🔗 [pub.dev/packages/flutter_clean_starter](https://pub.dev/packages/flutter_clean_starter) · [GitHub](https://github.com/rajoanrahman100/flutter_clean_starter)
